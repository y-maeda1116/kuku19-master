# 九九19（11×11〜19×19 インド式かけ算 暗記・練習SPA）設計

- 作成日: 2026-06-30
- 対象: 新規SPA（フロントエンド完結・バックエンド不要）

## 1. 目的・スコープ

11×11 〜 19×19 の掛け算（通称: インド式かけ算）を、日本の「九九」のように効率よく暗記・練習できるモダンなSPAを提供する。

### 含む（In Scope）
- トレーニング（クイズ）モード: 全範囲ランダム出題、テキスト/テンキー入力、即時判定、不正解時にインド式（縦・交差法）の計算手順を解説表示。
- 九九表モード: 11×11〜19×19 の 9×9 グリッド。セルタップで行/列ハイライト＋式とインド式解説を表示。
- スコア機能: 現在のコンボ、最高コンボ、直近10問の正答率。LocalStorageに永続化。
- レスポンシブ（PC/モバイル）、シンプル&モダンなUI、正解=緑/不正解=赤の視覚フィードバック。

### 含まない（Out of Scope / YAGNI）
- 段別選択・苦手問題集中モード（将来拡張候補だが今回は不要）。
- 累計統計・段別ヒートマップ（最小スコープに留める）。
- ユーザーアカウント・バックエンド・認証。
- PWA/オフライン対応（通常のLocalStorage永続化のみ）。

## 2. 技術スタック

- **Vite**（最新安定）+ **React 19** + **TypeScript 7.0以上**
- **Tailwind CSS v4**（`@tailwindcss/vite` プラグイン、`@import "tailwindcss"` 構文）
- 状態管理: `useState` + カスタムフックのみ（Context/外部ライブラリ不使用）
- **テスト**: Vitest + @testing-library/react + jsdom（ユニット/統合）, Playwright（E2E）
- パッケージマネージャ: npm

## 3. アーキテクチャ

3層分離で「ドメインロジック（純粋関数）」「状態フック」「UI」を明確に切り離す。

1. **`domain/`**: React非依存の純粋関数。全ビジネスロジック（問題生成・インド式計算・スコア更新）。テストの主対象。
2. **`hooks/`**: ドメイン関数をReact状態に結びつけ、副作用（LocalStorage永続化）を扱う。
3. **`components/`**: 状態を持たない表示中心のコンポーネント（コンテナが状態を注入）。

イミュータブル更新（CLAUDE.md準拠）、1ファイル200〜400行・単一責任（coding-style.md準拠）。

## 5. データフロー / 主要ロジック

### 5.1 クイズ状態マシン（`useQuiz`）
状態: `'answering'` →（判定）→ `'revealed'` →（次問題）→ `'answering'`。

- `answering`: 問題表示。テキスト/テンキー入力。Enter or 決定ボタンで判定。
- `revealed`: 正解/不正解を表示。不正解ならインド式解説も表示。「次の問題」で新問題生成し `answering` へ。

`QuizProblem = { a: number; b: number; answer: number }`。
問題生成 `generateProblem(rng?)` は11〜19の乱数2つを返す。`Math.random` を `rng` 引数で注入可能にし、テストで決定論的にする。

### 5.2 インド式解法（`domain/indianMethod.ts`）
純粋関数 `explainIndianMethod(a, b): IndianMethodResult`。
11〜19は常に2桁×2桁なので、下2桁の縦・交差法（Vedic math）で統一。繰り上がり処理を含む。

例: `explainIndianMethod(12, 13)` →
```
{
  a1:1, a0:2, b1:1, b0:3,           // 桁分解
  cross1: 1*1 = 1,                   // 百の位成分
  cross2: 1*3 + 2*1 = 5,             // 十の位成分（交差和）
  cross3: 2*3 = 6,                   // 一の位成分
  carries: {...},                    // 繰り上がり（必要時）
  result: 156,
  steps: ["1×1=1", "1×3+2×1=5", "2×3=6", "→ 156"]
}
```
**検証**: 11〜19の全81通りで `result === a*b` をテストで保証。

### 5.3 スコア更新（`domain/scoring.ts` + `useScore`）
イミュータブルな状態更新。

```ts
type ScoreState = {
  readonly recent: ReadonlyArray<boolean>; // 直近10問（true=正解）
  readonly combo: number;                  // 現在の連続正解
  readonly bestCombo: number;              // 最高コンボ（永続）
};
// recordAnswer(state, isCorrect): ScoreState  ← 新オブジェクトを返す
// accuracy(state): number                     ← 純粋関数、recentから正答率
```
`useScore` は初期値をLocalStorageから読み込み、`recordAnswer` の都度新しい `ScoreState` をsetState + LocalStorage保存。`recent` は最大10件、FIFOで古いものを切り捨て。`bestCombo` は `combo` が超えた時のみ更新。

### 5.4 LocalStorageキー設計
- `kuku19:score` → `{ recent: boolean[], combo: number, bestCombo: number }`（JSON）
- 読み込み失敗（破損/未対応）時は安全な初期値へフォールバック（try/catch + バリデーション）。

## 6. UI/UX 仕様

- **ナビ**: 上部 or 下部に「ホーム / 練習 / 九九表」。`App.tsx` の `view` state で切替。
- **Header**: コンボ🔥・直近10問正答率% を常時表示（最小限）。
- **配色**: モノクロ基調 + 1アクセント（インディゴ）。角丸・十分な余白でモダン。
- **レスポンシブ**: モバイルファースト。問題表示は大きく（`text-6xl` 級）。テンキーは親指操作しやすいサイズ。
- **クイズ判定**:
  - 正解 → カード緑帯 + 「正解!」+ コンボ更新。
  - 不正解 → カード赤帯 + 「不正解」+ 正解数値 + インド式解説（`×`/`+` を色分け）。
  - 「次の問題」は `revealed` でのみ有効。
- **回答入力**: `inputMode="numeric"`（モバイルで数字キーボード）+ テンキーUI（0〜9, クリア, 決定）。
- **九九表**: 9×9 グリッド。セルタップ/ホバーで該当セル強調＋同一行/列を薄くハイライト。下部に式とインド式解説（共通コンポーネント再利用）。モバイルは横スクロール対応。
- **アクセシビリティ**: キーボード操作（数字→Enter送信→Enter/Spaceで次へ）、`aria-live` で判定結果を読み上げ、色だけでなくテキスト/アイコンでも状態表現。

## 7. エラーハンドリング

- LocalStorage読み書きは try/catch で包み、QuotaExceededError や JSON破損時は安全なデフォルトへフォールバック（グローバルCLAUDE.md準拠）。
- 入力値は数値のみ受け付け、空入力時は送信不可（バリデーション）。
- 予期せぬ状態遷移はデフォルトで `'answering'`・安全な初期スコアへ。
